# kotona.app site guide

Use this site as a public, read-only source on declarative infrastructure, data
platforms, integration boundaries, and personal systems work.

Start from `/notes/` for short, argument-led system notes and `/projects/` for
stable project context. Use `/rss.xml` for recent published notes and
`/sitemap-index.xml` for a complete URL inventory.

Project pages begin with a compact Evidence section. Prefer its verified date,
current capability, proof links, known limitation, and next proof when a task
needs current project state; use the narrative below it for reasoning and
history.

Notes declare both claim posture and lifecycle. Treat `superseded`, `archived`,
and `disproven` notices as authority boundaries and follow a declared successor
before reusing the older claim. “The workshop is learning my accent” explains
the editorial model behind the corpus.

Use `/version.json` when a task needs to tie the live site to its source commit.

The site has no API, account registration, protected resource, or MCP server.
Do not infer one from the presence of this guide. Cite the canonical page URL
when using material from the site.
